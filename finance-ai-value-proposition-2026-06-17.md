# AI 股票研究摘要助手：价值主张体验记录

日期：2026-06-17

使用技能：`value-proposition`

体验主题：AI 股票研究摘要助手

目标用户：AI PM 转行者 / 金融背景学习者

边界：本记录只讨论公开信息整理、风险提示和学习型研究摘要，不做荐股、不预测收益、不提供买入或卖出建议。

## 1. 为什么体验这个技能

前两份金融 AI 体验记录已经完成：

- `market-segments`：帮助拆出不同用户细分，并确认第一轮先服务自己这个“AI PM 转行者 / 金融背景学习者”。
- `competitor-analysis`：帮助对比 FinRobot、TradingAgents、ai-hedge-fund 和通用 LLM 方案，并识别它们分别属于研究摘要、交易模拟、教育 demo 或企业工作流。

这次体验 `value-proposition`，重点不是写一句漂亮口号，而是测试它能否把金融 AI 的价值主张从危险的：

> 帮你赚钱 / 帮你选股 / 帮你做投资决策

拉回到更安全、更适合当前阶段的：

> 帮你理解公开信息、降低研究成本、识别风险和训练 AI PM 产品判断。

## 2. 六部分价值主张

### 2.1 服务谁

第一轮服务对象是：

> 正在转向 AI 产品经理、并且有一定金融 / 数据背景，但没有丰富股票投资经验的人。

这个用户的特点：

- 想通过真实热门方向训练 AI PM 判断。
- 能理解金融、数据和风险相关语言。
- 不想靠“预测收益”证明能力。
- 需要把 GitHub 项目、AI 产品和行业趋势转成作品集素材。
- 需要在高风险场景里练习边界意识和证据意识。

这个用户不是：

- 希望 AI 直接推荐股票的普通投资者。
- 专业基金经理或投研分析师。
- 量化交易工程师。
- 想做自动交易的人。

## 2.2 为什么需要

核心问题不是“我该买哪只股票”，而是：

> 当我看到一个 AI 金融项目或投资助手产品时，我需要快速判断它是什么、服务谁、风险在哪里、哪些能力可复用到我的 AI PM 作品集。

这个用户想完成的 JTBD：

> 帮我从陌生但热门的 AI 金融项目中提取产品理解，而不是让我陷入技术名词、投资术语和不可靠结论里。

具体目标：

- 快速看懂一个 AI 金融项目的定位。
- 分清它是研究摘要、投资分析、交易模拟、教育 demo，还是企业工作流。
- 识别它有没有把推断伪装成事实。
- 看出它是否有来源、边界、免责声明和风险提示。
- 把研究结果转成 GitHub 作品集记录。

## 2.3 使用前状态

现在的做法通常是：

- 自己打开 GitHub README。
- 看项目介绍、star、demo 图和文档。
- 用 ChatGPT / Claude 问“这个项目是什么”。
- 自己手动整理成笔记。
- 再尝试判断它对 AI PM 有没有启发。

摩擦点：

- README 经常偏技术，外行很难判断产品价值。
- 项目名字可能很吸引人，例如 hedge fund、trading agents，但真实边界不清楚。
- 通用 LLM 会给出顺滑回答，但不一定区分事实和推断。
- 金融场景风险高，容易把“研究摘要”误解成“投资建议”。
- 笔记容易停留在资料摘抄，没有变成产品判断。

## 2.4 如何解决

AI 股票研究摘要助手应该提供一个固定工作流：

1. **项目类型判断**  
   先判断对象属于研究摘要、投资分析、交易模拟、教育 demo、企业工作流中的哪一类。

2. **用户和场景拆解**  
   说明谁会用、在什么场景用、目标是什么。

3. **事实 / 推断 / 待验证分层**  
   把公开资料能确认的内容、基于资料的推断、必须真实体验后才能判断的问题分开。

4. **风险和边界提示**  
   明确标注不做荐股、不预测收益、不替代专业投资判断。

5. **竞品和替代方案对比**  
   对比 FinRobot、TradingAgents、ai-hedge-fund、通用 LLM 等不同形态。

6. **作品集转化建议**  
   告诉用户这次分析可以如何写入 GitHub 作品集，例如用户细分、竞品对比、MVP PRD、red team。

它的关键不是“生成更多内容”，而是让内容更安全、更可复盘、更像产品经理的判断。

## 2.5 使用后状态

使用后，用户应该能得到：

- 一份结构化研究摘要。
- 一个明确的项目类型判断。
- 一张事实 / 推断 / 待验证表。
- 一个风险边界说明。
- 一个对 AI PM 的启发总结。
- 一个下一步体验任务清单。
- 一个可以放入 GitHub 的作品集文档初稿。

用户的变化：

- 从“看不懂热门项目”变成“能拆出用户、场景、风险和边界”。
- 从“泛泛问 AI”变成“用固定框架做产品分析”。
- 从“怕自己不懂投资”变成“知道我分析的是 AI 产品边界，不是投资收益”。

## 2.6 替代方案

| 替代方案 | 优点 | 缺点 |
| --- | --- | --- |
| 直接用 ChatGPT / Claude | 灵活、快速、低成本 | 需要强提示词，不会默认区分事实 / 推断 / 待验证 |
| 手动读 README | 最接近原始资料 | 慢，容易只摘抄技术介绍，不形成产品判断 |
| FinRobot / TradingAgents 等项目文档 | 能学习真实项目 | 需要一定技术和金融背景，容易看不懂边界 |
| ChatPRD | 适合结构化文档 | 默认更偏 PRD 和产品文档，不一定适合 GitHub 项目事实核验 |
| Perplexity / NotebookLM | 来源和资料整理较强 | 不一定输出 PM 视角的作品集转化建议 |

这说明 AI 股票研究摘要助手如果要有价值，不能只是“再包装一个 ChatGPT prompt”。它必须默认提供：

- 事实分层。
- 风险边界。
- 体验任务清单。
- 作品集转化。

## 3. 简洁价值主张

面向正在转 AI PM、并希望用金融 AI 场景训练产品判断的人，AI 股票研究摘要助手可以把公开项目和产品资料整理成结构化研究摘要，帮助用户识别项目类型、风险边界和作品集机会，而不是提供投资建议或收益预测。

## 4. 定位语

> 给 AI PM 转行者用的金融 AI 项目阅读助手：帮你看懂项目、标注风险、沉淀作品集，不帮你荐股。

## 5. 价值主张的风险

这次 `value-proposition` 体验也暴露了几个风险：

### 风险 1：容易滑向投资建议

只要产品名字里有“股票”“投资”“研究”，用户就可能期待它告诉自己买什么。

必须在名称、README、界面和输出中反复强调：

- 不荐股。
- 不预测收益。
- 不替代专业投资判断。
- 只整理公开信息和风险。

### 风险 2：价值可能过窄

如果只服务我自己，外部商业价值暂时不清楚。

但当前阶段这不是问题。第一轮目标是服务转行学习和作品集，不是证明商业化。

### 风险 3：对金融知识仍有要求

即使只是研究摘要，用户仍然需要理解一些基础概念：

- 财报。
- 公司业务。
- 风险因素。
- 数据来源。
- 市场情绪。
- 合规边界。

所以产品不能假设用户完全零基础，需要提供术语解释和“我还不懂什么”的提示。

## 6. 本次技能体验评价

### 有用之处

- `value-proposition` 的 6 部分结构有助于避免一上来写功能。
- 它迫使我先说清楚服务谁，而不是泛泛服务所有投资者。
- “替代方案”部分很有价值，因为它提醒我最大竞品其实是通用 LLM + 好提示词。
- 它帮助把价值主张从“赚钱”纠正成“理解、降本、控风险、做作品集”。

### 不足之处

- 如果没有人工边界控制，这个技能很容易写出过度营销的价值主张。
- 它不会天然知道金融合规风险，需要在输入里明确强调。
- 它仍然需要结合真实体验和用户反馈验证。

### 对 AI PM Research Agent 的启发

以后自己的工作流里，价值主张模块应该默认加入：

1. **危险价值主张排除**  
   例如金融场景排除“帮你赚钱”，医疗场景排除“替你诊断”。

2. **安全价值主张重写**  
   把高风险承诺改写成辅助、解释、整理、提示风险。

3. **第一用户限定**  
   先服务一个具体用户，而不是所有人。

4. **最大替代方案识别**  
   默认问：为什么不用 ChatGPT / Claude + 提示词？
