# competitor-analysis 技能体验边界记录

日期：2026-06-17

体验技能：`competitor-analysis`

体验主题：AI 股票研究摘要助手

## 1. 这次发现了什么

在复跑 `competitor-analysis` 后，我发现一个关键问题：

> 这个技能能生成结构化竞品分析，但它并不会真的像用户一样去体验产品。

它更擅长的是基于公开资料做分析，例如：

- README
- 官网介绍
- 产品描述
- 定价页
- 公开文章
- GitHub 项目信息
- 用户评论或社区讨论

但传统产品经理做竞品分析时，通常不只是“看资料”，还会亲自像用户一样体验：

- 注册账号。
- 走新手引导。
- 输入真实任务。
- 看每一步交互是否顺畅。
- 观察免费版限制。
- 测试导出、保存、分享等功能。
- 记录截图。
- 比较输出质量。
- 感受哪里卡、哪里好用、哪里误导。

所以，`competitor-analysis` 不能等同于完整竞品体验。

## 2. 这个技能实际帮到了什么

它主要帮我完成三件事：

### 资料整理

它可以快速把几个项目或产品放在同一个框架里比较。

例如这次它帮助整理：

- FinRobot 更像金融研究报告自动化工具。
- TradingAgents 更像多 Agent 交易公司模拟框架。
- ai-hedge-fund 更像教育用途的投资 Agent demo。
- ChatGPT / Claude 强提示词方案是最容易上手的通用替代方案。

### 分类判断

它能帮我避免把所有热门项目混在一起。

同样都叫 AI 金融项目，但它们可能分别属于：

- 研究摘要。
- 投资分析。
- 交易决策。
- 教育 demo。
- 企业工作流。

这个分类对 AI PM 很重要，因为不同类型对应完全不同的用户、风险和边界。

### 问题清单

它能帮我生成下一步要验证的问题，例如：

- 这个项目有没有清楚声明非投资建议？
- 它是否标注信息来源？
- 它是否区分事实和推断？
- 它是否需要安装和配置数据源？
- 它是否能跑出真实 demo？
- 它的输出是否稳定？

## 3. 它没有帮到什么

### 没有真实体验产品

它不会自动帮我打开产品、注册账号、跑 demo 或截图。

如果我要判断产品体验，就必须自己或通过浏览器/终端工具真实操作。

### 没有验证输出质量

它能说“这个产品可能适合某类用户”，但不能证明这个产品真的好用。

例如它可以分析 FinRobot 的定位，但不能仅凭分析判断：

- 安装是否顺利。
- 数据源是否可用。
- 生成报告是否准确。
- 报告是否有幻觉。
- 输出是否能给真实用户使用。

### 容易把公开资料当成真实能力

如果 README 写得很漂亮，`competitor-analysis` 可能会把它整理成一个看起来很完整的产品分析。

但实际产品可能存在：

- 安装门槛高。
- demo 跑不通。
- 免费数据源不可用。
- 输出质量不稳定。
- 文档和真实体验不一致。

这些只有真实体验后才知道。

## 4. 我对这个技能的重新理解

`competitor-analysis` 更适合做：

> 竞品资料分析和体验前的问题清单。

不适合直接替代：

> 真实用户体验和可用性测试。

更准确的竞品分析应该分成三层：

| 层级 | 任务 | 谁更擅长 |
| --- | --- | --- |
| 资料层 | 看 README、官网、定价页、介绍文章，生成竞品对比 | `competitor-analysis` |
| 体验层 | 注册、安装、跑 demo、截图、测试输出质量 | 人 / 浏览器 / 终端 |
| 判断层 | 把资料和真实体验结合，判断是否适合我的作品集 | 我 + Codex |

## 5. 以后应该怎样使用这个技能

以后使用 `competitor-analysis` 时，应该明确要求它区分三类信息：

```text
请区分：
1. 公开资料能确认的事实
2. 基于资料的合理推断
3. 必须真实体验后才能判断的问题
```

还应该让它输出“真实体验任务清单”：

```text
请告诉我，如果我要像真实用户一样体验这些竞品，应该分别做哪些操作？
```

例如分析 FinRobot 时，不能只问“它是什么”，还要问：

- 如何安装？
- 需要哪些 API key 或数据源？
- 能否用一个 ticker 跑通 demo？
- 输出报告是否包含来源？
- 是否有免责声明？
- 是否会输出买卖建议？
- 输出中哪些是事实，哪些是推断？

## 6. 对 AI PM Research Agent 的启发

如果未来沉淀自己的工作流，竞品分析模块不能只输出“竞品表”。

它必须默认包含：

1. **资料分析**  
   基于公开资料总结竞品定位、用户、场景和差异。

2. **事实 / 推断 / 待验证分层**  
   避免把公开文案直接当成真实能力。

3. **真实体验任务清单**  
   告诉用户下一步应该如何自己验证。

4. **体验后复盘模板**  
   记录真实使用过程中的卡点、限制、截图、输出样例和主观感受。

5. **产品判断**  
   结合资料层和体验层，判断这个竞品对作品集有什么启发。

## 7. 本次结论

这次最重要的收获不是某个金融 AI 项目的分析，而是理解了一个技能边界：

> AI 生成的竞品分析不是竞品体验。它适合帮我快速搭框架、列问题、做初筛，但真正的产品判断仍然需要真实使用和人工复盘。

这个发现应该成为后续所有 skills 体验的基本原则。
